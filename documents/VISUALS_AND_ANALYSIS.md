# Visuals and Analysis Documentation

This document describes the data analysis outputs and visualizations produced by the `not-an-llm` repository for collected datasets.

## Pipeline overview

The analysis pipeline separates three stages:

1. `collect`
   - Fetches paper metadata and text from Semantic Scholar, arXiv, medRxiv, or external pair datasets.
2. `preprocess`
   - Cleans raw text, extracts normalized text fields, and saves preprocessed JSONL for feature analysis.
3. `analyze` / `visualize`
   - Computes style and readability features, aggregates them over time, and then generates plots from aggregated trends.

The `analyze` step can produce both analysis artifacts and plots, while the `visualize` step regenerates plots from precomputed trend CSVs.

## Data artifacts produced

### Preprocessing output

- `data/analyzed/<dataset>.jsonl`
  - Preprocessed records with normalized text fields (`text_clean`, `text_lemma`, etc.) and metadata.

### Feature analysis output

- `data/analyzed/<dataset>_features.jsonl`
  - Document-level feature-enriched JSONL output from the analysis pipeline.
  - Includes extracted style features, syntactic counts, discourse markers, readability metrics, and dependency statistics.

### Trend CSV outputs

- `data/analysis/<dataset>_trends_by_year.csv`
  - Yearly aggregated feature means, standard deviations, and counts.

- `data/analysis/<dataset>_trends_by_month.csv`
  - Monthly aggregated feature means for the same features.

- `data/analysis/feature_stats.csv`
  - Statistical summary computed from the yearly trends.
  - Includes pre/post comparisons, t-tests, Cohen's d, regression slopes, and change point estimates.

### External comparison outputs

For paired external datasets such as HC3 or MAGE:

- `data/external/..._features.jsonl`
  - Feature-enriched paired records for human and AI text.

- `data/external/..._comparison.csv`
  - Mean and standard deviation for each feature by source group.
  - Includes AI-human difference, ratio, and Cohen's d.

## Analysis feature categories

The analysis pipeline extracts writing-style features from preprocessed text and metadata. The main feature categories are:

### 1. LLM-style marker and discourse features

- Marker words
- Marker verbs
- Marker adjectives
- Marker phrases
- Sequential markers
- Causal markers
- Contrast markers
- Emphasis markers
- Summary markers
- Hedging terms
- Certainty terms

These features are used as token-based counts and rates per 1,000 words.

### 2. Syntax and dependency features

- Dependency entropy
- Dependency length
- Coordination count
- Clause depth
- Sentence depth and clause depth statistics
- Dependency distribution by tag
- List-of-three coordination markers

### 3. Readability metrics

When enabled, the pipeline computes readability statistics such as:

- Average words per sentence
- Average syllables per word
- Flesch Reading Ease
- Flesch-Kincaid Grade
- Dale-Chall score

### 4. Document metadata and normalization

- Word count
- Sentence count
- Publication year / month
- Cleaned and lemmatized text fields

## Visualization outputs

All plots are saved to `data/visuals/<dataset_stem>/` or the configured `trends_plot_dir`.

### Core plots generated by `run_visualization`

1. `*_mean.png`
   - One plot per numeric feature.
   - Shows yearly mean values, optionally with monthly trend lines.
   - Annotated with LLM-related event lines.

2. `pre_post_diff.png`
   - Global pre/post comparison for all features.
   - Uses symmetric percent change between pre-LLM and post-LLM periods.
   - Visualizes features sorted by effect size.

3. `*_diff.png` / grouped feature diff plots
   - Per-feature-group diff plots for feature sets defined in `FEATURE_GROUPS`.
   - Examples: `marker_words_diff.png`, `verbs_diff.png`, `adjectives_diff.png`, `phrases_diff.png`, `syntax_diff.png`, `readability_diff.png`.

4. `*_trend.png` group plots
   - Trend plots for marker groups such as marker words, sequential markers, causal markers, contrast markers, emphasis markers, and summary markers.
   - Each group shows mean counts per 1,000 words over time.

5. `stacked_trends.png`
   - Stacked layout of yearly feature trends for multiple features in one figure.
   - Useful for multi-feature visual comparison across years.

6. `word_prefix_stack.png`
   - Vertical stack of all `word_` prefix feature trends.
   - Each feature gets its own y-axis while sharing the same ChatGPT event line for comparison.

7. `verb_prefix_stack.png`
   - Vertical stack of all `verb_` prefix feature trends.
   - Each feature gets its own y-axis while sharing the same ChatGPT event line for comparison.

8. `adjective_prefix_stack.png`
   - Vertical stack of all `adjective_` prefix feature trends.
   - Each feature gets its own y-axis while sharing the same ChatGPT event line for comparison.

9. `readability_stack.png`
   - Vertical stack of all readability features (avg_words_per_sentence, avg_syllables_per_word, flesch_reading_ease, flesch_kincaid_grade, dale_chall).
   - Each feature gets its own y-axis while sharing the same ChatGPT event line for comparison.

11. `dependency_distribution_diff.png`
    - Change plot showing the top dependency roles by proportional change before vs after 2023.

12. `dependency_distribution_trends.png`
    - Line chart showing yearly changes for the top dependency roles selected by their pre/post change.

### Additional plot generation in `analyze`

When `generate_plots = true`, the `analyze` pipeline can also generate the same trend plots during analysis and save them directly alongside trend CSVs.

## Statistical analysis performed

The trend analysis pipeline computes the following statistical summaries from yearly trend data:

- Mean difference between pre-LLM and post-LLM periods
- T-tests for pre/post year groups
- Cohen's d effect size
- Linear regression slope across years
- Slope p-value from Pearson correlation
- Estimated change point using the `ruptures` library
- Multiple testing corrected p-values (Benjamini/Hochberg-style adjustment)

## External human-vs-AI comparison analysis

The paired external analysis pipeline produces a comparison table and a top-difference plot for features that separate human-written from AI-written text.

- `human_vs_ai_top_differences.png`
  - Shows strongest positive and negative feature differences between AI and human text.

## Configurable behavior

Plot and analysis behavior is controlled in `config.toml`, `config_mini.toml`, and `config_external.toml`.

Key settings include:

- `analysis.enabled`
- `analysis.features`
- `analysis.readability_metrics`
- `analysis.syntactic_features`
- `analysis.generate_plots`
- `analysis.trends_plot_dir`
- `analysis.llm_marker_words` / `marker_verbs` / `marker_adjectives` / `marker_phrases`
- `analysis.sequential_markers` / `causal_markers` / `contrast_markers` / `emphasis_markers` / `summary_markers`
- `analysis.hedge_terms` / `analysis.certainty_terms`

## Usage notes

- Run `analyze` to compute feature datasets and trend CSVs.
- Run `visualize` to regenerate plots from existing trend CSVs.
- Use the `external-analyze` pipeline for non-temporal, paired human/AI comparison datasets.
